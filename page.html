SnapMap (ongoing)

  • This application takes an upload of snapchat metadata packages and renders them in an interface making them more consumable for users to make use of their data outside of the main snapchat application. User Login and Registration forms are configured to send api requests to trigger the respective backend database interactions. The experience pipeline will flow as such:
    • Creating a User and Logging In
    • Uploading Metadata
    • Choosing What Collection to Render to the Interface

This project will push me as a developer and require more research. The metadata packages will contain some pretty high resolution photos and videos. I am thinking of using AWS to store this sensitive data in S3 buckets, then storing direction to those resources in my database instead of trying to store actual files as bit data. I am concerned that this may be a weakness in this application, but it is the only method I can conceive of at the moment and seems to be backed up by my research. Unfortunately, there doesn't seem to be an Snapchat API that will allow users to pull their metadata directly from the app, and I have to assume that users will understand how to upload to the application. Luckily, I am still building out that portion of the backend, I think I can be less strict about validating uploads so that whatever is uploaded is handled appropriately.
